# Переменные макросов — синтаксис `{{variable}}`

## Базовый синтаксис

Используйте `{{имя_переменной}}` для подстановки значений в полях шагов.

## Поддерживаемые поля

| Поле | Описание | Пример |
|------|----------|--------|
| `value` | Значение для ввода, set-variable | `{{email}}` |
| `cssSelector` | CSS-селектор элемента | `{{selector_name}}` |
| `url` | URL для навигации | `https://example.com/{{page}}` |
| `compareValue` | Значение для сравнения в if | `{{expected}}` |
| `count` | Количество итераций цикла | `{{loop_count}}` |
| `tableName` | Имя таблицы | `{{table}}` |
| `varName` | Имя переменной-назначения | `{{target_var}}` |
| `rowIndex` | Индекс строки таблицы | `{{row}}` |
| `columnName` | Имя колонки таблицы | `{{col}}` |
| `promptTitle` | Заголовок запроса ввода | `Введите пароль для {{email}}` |
| `key` | Клавиша для нажатия | `{{key_name}}` |

## Фильтры

Фильтры применяются через `|` после имени переменной:

| Фильтр | Описание | Пример |
|--------|----------|--------|
| `numbers_only` | Оставить только цифры | `{{phone\|numbers_only}}` → `79001234567` |
| `trim` | Убрать пробелы по краям | `{{text\|trim}}` → `hello` |

## Встроенные переменные

| Переменная | Описание |
|------------|----------|
| `{{_current}}` | Текущий элемент в `loop-elements` (используется как селектор) |
| `{{_loop_index}}` | Индекс текущей итерации (начиная с 0) |
| `{{_loop_total}}` | Общее количество итераций |

## Типы переменных

### Из таблицы (loop-table)
- **Автоматически создаются** при использовании блока `loop-table`
- Каждая **колонка таблицы** становится переменной
- Доступны внутри цикла и во вложенных блоках

**Пример:**
```
Таблица "accounts":
| username | password | email              |
|----------|----------|--------------------|
| user1    | pass123  | user1@example.com  |
| user2    | pass456  | user2@example.com  |

🔄 Цикл по таблице "accounts"
  ├── 📌 Клик → #username-field
  ├── ✍️ Ввод → {{username}}        ← итерация 1: "user1", итерация 2: "user2"
  ├── 📌 Клик → #password-field
  ├── ✍️ Ввод → {{password}}        ← итерация 1: "pass123", итерация 2: "pass456"
  └── 📌 Клик → .login-button
```

### Эфемерные (runtime)
- Создаются через `set-variable`, `read`, `read-current`
- Живут только во время выполнения макроса
- Исчезают после завершения

### Глобальные
- Задаются в Настройки → Переменные
- Доступны во всех макросах
- Сохраняются в `data/settings.json`

### Персистентные
- Отмечены флагом `persistent: true`
- Сохраняются в `data/variables/persistent.json`
- Переживают перезапуск сервера и макросов
